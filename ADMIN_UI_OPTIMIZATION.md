## 💡 设计思路

### 为什么这样改？

1. **安全性优先**
   - App Secret 是敏感信息，不应该随意显示
   - 即使用户有权限查看，也应该通过主动操作（点击复制）来获取

2. **减少视觉干扰**
   - 应用列表不需要显示 App Secret
   - 需要时可以通过详情查看（虽然是掩码）

3. **提高操作效率**
   - 测试场景移到卡密页面，更符合使用场景
   - 用户在测试卡密功能时，通常需要同时打开测试场景

4. **保持一致性**
   - 两个页面的 App Secret 都使用相同的掩码显示方式
   - 复制功能保持一致

## 📝 使用说明

### 查看 App Secret

**方法 1：通过卡密页面复制**
1. 进入"卡密管理"页面
2. 选择目标应用
3. 在凭证栏找到 "App Secret" 行
4. 点击右侧的"复制"按钮
5. App Secret 已复制到剪贴板

**方法 2：通过应用详情（仅查看掩码）**
1. 进入"应用列表"页面
2. 点击应用的"详情"按钮
3. 在详情对话框中可以看到 App Secret 显示为 `••••••••••••••••`
4. ⚠️ 注意：此处无法复制真实值

### 打开测试场景

1. 进入"卡密管理"页面
2. 选择目标应用
3. 在凭证栏底部点击"测试场景"按钮
4. 系统会在新标签页打开测试页面
5. 测试页面会自动填入当前应用的 app_id

## ⚠️ 注意事项

1. **App Secret 的安全性**
   - 虽然界面上隐藏了 App Secret，但它仍然存储在数据库中
   - 确保只有授权用户可以访问管理后台
   - 建议定期轮换 App Secret

2. **复制功能的依赖**
   - 复制功能依赖浏览器的 Clipboard API
   - 已实现降级方案，兼容旧版浏览器
   - 如果复制失败，会提示用户手动复制

3. **测试场景的访问**
   - 测试场景页面位于 `/sdk/js_example.html`
   - 需要确保该文件存在于 public 目录
   - 测试页面通过 URL 参数接收 app_id
